# hachi-position Landscape Research

Use current sources for position reports because vendors, OSS projects, and product categories change quickly.

## Source Order

1. Source SEED evidence from hachi-sense.
2. Official docs, release notes, product pages, and GitHub repositories.
3. GitHub Issues and Discussions for unresolved pains.
4. Commercial alternative documentation and pricing pages.
5. Community posts, reviews, and benchmarks.
6. Analyst or consulting reports only as background, not as sole evidence.

## Big Vendor Checks

Look for adjacent features from:

- OpenAI
- Anthropic
- Google
- Microsoft / GitHub
- Vercel
- AWS
- Cloudflare
- Atlassian
- Notion
- Linear
- Datadog
- JetBrains
- Docker

Ask:

- Is this becoming a platform feature?
- Is the value mostly inside a vendor's SDK or dashboard?
- Can the idea survive as a companion workflow, local tool, Skill, CLI, or review layer?
- What niche would the vendor underserve?

## Alternative Categories

Check these categories when relevant:

- Commercial SaaS
- OSS frameworks
- CLI tools
- IDE extensions
- GitHub Actions / CI tools
- Notion / Slack / Jira / Linear workflows
- Manual spreadsheets and custom scripts

## Evidence Hygiene

- Provide source links for non-obvious claims.
- Separate observed facts, inferences, and judgments.
- Do not overstate market size from anecdotes.
- If evidence is thin or inaccessible, lower confidence and recommend `watch`.
- Before final scoring, read `research-completion.md` and state any research gaps.
