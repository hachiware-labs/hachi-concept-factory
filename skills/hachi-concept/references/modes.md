# hachi-concept Modes

## Extract Mode

Use for repositories, README files, docs, issues, examples, CLI help, package descriptions, code structure, and rough notes.

Inspect:

- README and docs
- package description
- examples and CLI usage
- API surface
- directory structure
- issues and discussions
- release notes
- tests
- AGENTS.md, CLAUDE.md, SKILL.md, or other agent-facing docs

Ask:

- What does this enable?
- Who has the pain?
- What is the latent product category?
- What current alternatives exist?
- What would become the first Skill, CLI, SaaS, or UI wedge?

## Refine Mode

Use for one concept, one SEED, or one positioning brief.

Refine:

- Target user
- First context
- Pain and current workaround
- Product form
- Differentiation
- Moat
- Wedge
- MVP direction
- Non-goals
- PRD readiness

## Synthesize Mode

Use for multiple SEEDs, positioning briefs, repositories, or past concept notes.

Synthesize by:

- Finding the common pain or workflow
- Dropping weaker overlapping ideas
- Choosing a first wedge
- Preserving useful rejected paths as future options or SEEDs
- Producing one coherent concept, not a bundle of features
